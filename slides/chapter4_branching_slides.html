4.1 The Branch Concept COP 1047C · MDC

Core Idea

What Is a Branch?

A branch is a sequence of statements that only executes under a certain condition. Programs make decisions by evaluating expressions that are True or False.
  • An if branch executes only when its expression is True
  • An if-else branch always executes one of two paths
  • An if-elif-else chain handles three or more outcomes
  • Exactly one branch in an if-elif-else executes per run
Real-world analogy: A restaurant host seats a party of 1 at the counter, party of 2 at a small table, and larger parties at a large table — one rule fires per group.

4.2 Detecting Equal Values with Branches COP 1047C · MDC

Python if, if-else, elif Syntax

# if statement
if num_years == 50:
    print("Congrats!")
    hotel_rate = hotel_rate / 2
# if-else statement
if div_remainder == 0:
    print(f"{user_num} is even.")
else:
    print(f"{user_num} is odd.")
# multi-branch elif
if num_years == 1:
    print("First year!")
elif num_years == 25:
    print("Silver!")
elif num_years == 50:
    print("Golden!")
else:
    print("Nothing special.")
🔑 Use == (equality) to compare, never = (assignment) inside a condition.
📐 Indentation is required. Branch statements must be indented (4 spaces standard).

⚠ COMMON ERROR Writing if x = 5: causes a SyntaxError. Always use == for comparison.

⚠ INVALID OPERATORS =>, !<, <> are not valid Python operators. Python uses >=, <=, !=. 
user_age = int(input("Enter age: "))

if user_age < 16:    # ≤ 15

Once a branch is entered, the remaining elif/else branches are skipped. Each branch's range is implicitly bounded by the previous False expressions.
4.4 Operator Chaining COP 1047C · MDC

Operator Chaining

Python lets you chain relational operators left-to-right, just like math notation.

# Is user_grade in high school?
if 9 <= user_grade <= 12:
    print("in high school")
else:
    print("not in high school")
9 <= user_grade <= 12 is equivalent to
(9 <= user_grade) and (user_grade <= 12)
How Chaining Works Evaluates left-to-right. If a < b is False, evaluation stops — b < c is never checked.
Examples 0 <= x < 100 — x is a nonneg. number under 100

z < x < y — x is between z and y

10 < x < 15 — x is 11, 12, 13, or 14

4.5 / 4.6 Ranges with Logical Operators COP 1047C · MDC

Ranges With & Without Gaps

No Gap — Implicit Ranges

Sequential elif is cleaner when ranges are contiguous — the lower bound is implicit.

if x < 0:
    # negative
elif x <= 10:
    # 0..10  (implicit x >= 0)
elif x <= 20:
    # 11..20 (implicit x > 10)
else:
    # 21+
With Gap — Explicit Ranges

Use and/or when ranges have a gap (non-contiguous values).

# Movie ticket — gap is ages 13–64
if user_age <= 12:
    ticket = 11   # child
elif user_age >= 65:
    ticket = 12   # senior
else:
    ticket = 14   # adult

# Office with two valid ranges:
if (office_num >= 100 and
        office_num <= 150) or \
   (office_num >= 200 and
        office_num <= 250):
    print("Valid office")

4.7 Multiple & Nested if Statements COP 1047C · MDC

Multiple if vs. Nested if-else

Multiple Independent if Statements

Each if is independent — more than one can execute.

# ALL that are True will execute!
if cash > 12:
    print("Buy a hat.")

if cash >= 23:
    print("Buy a new coat.")

if cash <= 1:
    print("Save money.")
If cash = 25, the first two both print. There is no mutual exclusion.
Nested if-else

An if-else inside another branch — checks an additional condition only when the outer condition is True.

if user_choice == 2:
    if num_items < 0:
        print("choice 2, negative")
    else:
        print("choice 2, non-neg")
else:
    print("not choice 2")
💡 Key rule: if-elif-else → only one branch fires. Multiple separate if statements → any matching branch fires.
4.9 Membership & Identity Operators COP 1047C · MDC

Membership in / not in

OperatorMeaning
inTrue if left value is found in the right container
not inTrue if left value is NOT found in container
isTrue if both operands reference the SAME object
is notTrue if operands reference DIFFERENT objects
prices = ["$20", 15, 5]
print(15 in prices)   # True
print(44 in prices)   # False

# Works on strings too!
if "abc" in "123abcd":
    print("Found substring")

# Checking list membership
special_list = [-99, 0, 44]
if special_num in special_list:
    print("Special number")
in with dictionaries only checks keys, not values! 3 in {"A":1, "B":3} → False
⚠ AVOID is/is not for value comparison Use == to compare values. is checks memory identity, not equality — it can give unexpected results with integers, strings, etc.
Good Practice Use in / not in to replace long chains of or comparisons:

❌ x==1 or x==2 or x==3
✅ x in [1, 2, 3]

4.11 Code Blocks & Indentation COP 1047C · MDC

Code Blocks & Indentation

  • Python uses indentation (not braces) to define code blocks
  • Standard: 4 spaces per indentation level
  • A new code block follows any line ending with a colon (:)
  • Never mix tabs and spaces — Python 3 raises IndentationError
antique = False
domestic = False

if year < 1970:
    antique = True    # block 1

if model in ["Ford", "Chevy"]:
    domestic = True   # block 2

if antique:
    if domestic:         # nested!
        print("Old & domestic")
⚠ INDENTATION ERRORS These all cause errors or wrong behavior:

• Inconsistent indent within a block
• Mixing tabs and spaces
• Indenting code that shouldn't be in a block
• Missing indent after if/else/elif:
Tip: Configure Your Editor Set VS Code to insert 4 spaces when you press Tab. This prevents accidental tab/space mixing.
Long statements can wrap to the next line inside parentheses without breaking the code block — Python treats them as continuation lines.
4.12 Conditional Expressions COP 1047C · MDC

Conditional Expressions (Ternary)

Syntax:
value_if_true  if  condition  else  value_if_false

The condition in the middle is evaluated first. Then either the left or right expression is returned.

# Instead of:
if x < 100:
    y = 0
else:
    y = x

# Write:
y = 0 if (x < 100) else x

# More examples:
cond_str = "negative" if val < 0 else "non-negative"

num_users = num_users+1 if dir==3 else num_users-1
When to Use Best for simple value assignments where both branches just assign to the same variable. Avoid for complex logic — readability matters.
When NOT to Use If the two branches assign to different variables, a conditional expression isn't possible — use a regular if-else instead.
⚠ NOTE The condition is evaluated before left or right expressions — even though it appears in the middle syntactically.
4.8 Comparing Data Types & Common Errors COP 1047C · MDC

Comparing Data Types

  • Integers / floats: compared arithmetically. 5 <= 5.0 → True
  • Strings: compared character by character (Unicode values). "FRIDAY" == "friday" → False (case-sensitive!)
  • Lists / tuples: element-by-element; first mismatch determines result
  • Dicts: only == and !=; must match all keys AND values
  • Avoid == with floats — rounding errors make equality unreliable
my_str = "Tuesday"
print(my_str == "Tuesday")  # True
print(my_str == "tuesday")  # False!

# Comparing different types → TypeError
# 1 < "abc"  ← will crash!
⚠ COMMON BRANCHING ERRORS 1. Using = instead of ==:
  if x = 5: → SyntaxError

2. Invalid operators:
  =>, !<, <> are not valid

3. Comparing incompatible types:
  1 < "abc" → TypeError
String Comparison Note Most string comparisons use == or !=. Relational operators (<, >) compare by Unicode values — capital letters come before lowercase in the Unicode table.

SUMMARY Chapter 4 Wrap-Up COP 1047C · MDC

Chapter 4 Summary

Structure
  • if — one optional branch
  • if-else — exactly one of two paths
  • if-elif-else — exactly one of N paths
  • Multiple if — any/all can execute
  • Nested — if inside another branch
Operators
  • == != < > <= >= relational
  • and or not logical
  • in not in membership
  • is is not identity
  • Precedence: arithmetic → relational → not → and → or
Key Rules
  • Always == (never =) in conditions
  • 4-space indentation, never mix tabs/spaces
  • Use and for ranges, or for gaps
  • Parentheses make precedence explicit
  • Conditional expressions: x if cond else y
