Misconception 1 of 3 · 43% missed this

Which statement does NOT print a newline at the end?

print("Hello")
print("Loading...")
print("Done", end="")
print("Wait\n")
Type your answer in chat — A, B, C, or D — before we reveal
Reveal · print() newlines

The answer is C

✓ print("Done", end="") CORRECT

Every print() call adds a newline by default. You don't see \n in the string, because print tacks one on for you via the invisible default end="\n". To suppress it, you have to pass end="" explicitly.

The trap

43% of you looked at the strings for a \n. Wrong question. The right question is: "did this call override the default end?" Only C does.

print("A")          # newline after — default
print("B", end="")  # NO newline — explicit override
print("C", end="!") # "!" instead of newline
Misconception 2 of 3 · 43% missed this

Which single line is equivalent to this code?

if age < 18:
    x = x + 5
else:
    x = x + 1
x += 5 if age < 18 else 1
if age < 18 x = x + 5 else x = x + 1
x = 5 if age < 18 else x = 1
x += if age < 18: 5 else: 1
Type your answer in chat — A, B, C, or D — before we reveal
Reveal · The ternary expression

The answer is A

✓ x += 5 if age < 18 else 1 CORRECT

Python has a one-line conditional expression: VALUE_IF_TRUE if CONDITION else VALUE_IF_FALSE. The whole thing evaluates to one value — either 5 or 1 — which then gets added to x.

Read it like English

"Add 5 if age is under 18, else add 1." Same word order. That's not a coincidence — Python deliberately reads that way.

# Three equivalent forms:

# Form 1: full if/else (verbose, clearest)
if age < 18:
    x += 5
else:
    x += 1

# Form 2: ternary expression (compact)
x += 5 if age < 18 else 1

# Form 3: also valid — assign the chosen value
bonus = 5 if age < 18 else 1
x += bonus

You will see this pattern again in list comprehensions in a few weeks. Get comfortable with it now.

Reveal · Command-line anatomy

The answer is A

✓ Command: python3 · Arguments: analyze.py results.csv CORRECT

The command is the program you're running — the actual executable. Everything else on the line is an argument passed to that program. Here, python3 is the program; both analyze.py (the script) and results.csv (the data file) are arguments handed to it.

Why this matters

In a few weeks, you'll read command-line arguments inside Python using sys.argv. When you do, sys.argv[0] will be analyze.py and sys.argv[1] will be results.csv. If you think the script name is part of the command, the indexing will confuse you.

$ python3 analyze.py results.csv
  └──┬──┘ └────┬────┘ └────┬────┘
  command   argv[0]      argv[1]

Which call generates every odd number
between 11 and 21, including both?

range(11, 21, 2)
range(11, 22, 2)
range(11, 23, 2)
range(21, 11, 2)
This one had the lowest score — only 48% got it right. Type A, B, C, or D.
Reveal · range() with an inclusive upper bound

The answer is B

✓ range(11, 22, 2) CORRECT

The stop value in range() is exclusive — it's the first number that does not get included. We want 21 to be in our list, so we have to set stop to 22 (or anything from 22 up to 23). With step 2 from 11, the sequence stops before hitting 22.

Trace it:
range(11, 22, 2)
# 11 → include   (11 < 22 ✓)
# 13 → include   (13 < 22 ✓)
# 15 → include   (15 < 22 ✓)
# 17 → include   (17 < 22 ✓)
# 19 → include   (19 < 22 ✓)
# 21 → include   (21 < 22 ✓)
# 23 → STOP      (23 < 22 ✗)
The rule, plain English

If the problem says "include N", your stop value must be at least N+1. Some of you picked C (range(11, 23, 2)) — that also works! It produces the same sequence. But B is the minimum, so it's the canonical answer.

Why A is wrong: range(11, 21, 2) stops before 21, so the sequence is 11, 13, 15, 17, 19 — 21 is missing.
