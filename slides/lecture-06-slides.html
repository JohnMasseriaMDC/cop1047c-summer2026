Exam Technique✏️ "Playing Computer"

Every output-prediction question yields to the same method:

  1. Write the variables down in a table on your scratch paper
  2. Execute one line at a time, updating the table — never "eyeball" a loop
  3. Write each output the moment a print() runs
  4. Check the exit: what are the variables after the loop ends?
trace like this
count = 1
while count <= 3:
    print(count * 2)
    count += 1

#  count | test    | prints
#  ------+---------+-------
#    1   | 1<=3 T  |  2
#    2   | 2<=3 T  |  4
#    3   | 3<=3 T  |  6
#    4   | 4<=3 F  |  (exit)
# after the loop: count == 4

Chapter 1🧱 Foundations

  • Every program: input → process → output
  • Python is interpreted — statements execute one at a time (C++/Java are compiled)
  • input() always returns a string — convert with int() or float() before doing math
  • Types: int, float, str — check with type(x)
  • Comments start with #
  • Identifiers: letters, digits, underscores; can't start with a digit; case-sensitive
  • Errors: syntax (won't parse) vs. runtime (parses, then blows up)
ch1_essentials.py
name = input("Name: ")     # always a str
age = int(input("Age: "))  # now an int

print("Hi", name, sep="--")   # Hi--Ana
print("A", end="")            # no newline
print("B")                    # AB
print() named parameters: sep (between items, default space) and end (after the line, default \n).

drill_1.py
x = input()        # user types: 4

print(x + y)
B — 43. input() returns strings, and + on strings is concatenation: "4" + "3" is "43". To get 7, both inputs need int() around them. This single fact is worth multiple questions on the exam.

Chapter 2🧮 Variables & Expressions

  • = assigns, == compares. x = 5 stores; x == 5 asks and returns True/False
  • Case sensitivity: wage, Wage, WAGE are three different variables — misspelling creates a new one, silently
  • Floats are approximate: 0.1 + 0.2 is 0.30000000000000004 — never test float equality with ==; use abs(a - b) < 1e-9
  • Scientific notation: 6.02e23, 1.5e-10
  • Unicode: ord('A') → 65, chr(97) → 'a' — uppercase sorts before lowercase
ch2_essentials.py
check = 123.33
total = check * 1.20
print(f"Total: ${total:.2f}")
# Total: $147.99

# :.2f = format with exactly
# 2 decimal places (display only —
# the stored value keeps full precision)
Exam trap: 0.1 + 0.2 == 0.3 evaluates to False. If you see float equality in a question, be suspicious.

B. :.2f rounds to 2 decimal places — it doesn't truncate. 19.999 rounds to 20.00. (Option D is what you'd get with a plain string missing the f prefix — also a favorite exam variation.)

Chapter 3📘 Strings

  • Indexing starts at 0: in "Python", [0] is 'P', [5] is 'n'
  • Negative indexes count from the end: [-1] is the last character
  • len(s) = number of characters; the last valid index is len(s) - 1
  • Concatenation: + joins strings; * repeats them ("ab" * 3 → "ababab")
  • Strings are immutable — s[0] = "x" is a runtime error
  • Escape sequences count as ONE character: len("a\nb") is 3
ch3_strings.py
word = "Python"
print(word[0])        # P
print(word[-1])       # n
print(len(word))      # 6
print(word + "!")     # Python!

first = "Ana"
print(f"Hi {first}, {2 + 3} things")
# Hi Ana, 5 things  (any expression
# works inside the braces)

Three favorite exam traps live on this slide:
1. int(3.9) is 3 — truncation, not rounding.
2. int("3.5") crashes — int() can't parse a string with a decimal point.
3. 10 / 5 is 2.0, not 2 — regular division always produces a float, even when it comes out even.

Binary refresher: f"{10:b}" → "1010", and 0b1010 is 10. Know how to convert a small number by hand (8 + 2 = 1010).

What does this print?

drill_3.py
foods = ["pizza", "tacos", "sushi"]
foods.append("pasta")
print(foods[1])
print(foods[-1])
print(len(foods))
B. Index 1 is the second item (tacos) because indexing starts at 0. After the append, [-1] is the new last item (pasta), and the length is 4. If you picked A, you counted from 1 — the single most common indexing mistake.

Chapter 4🔀 if / elif / else

  • Anatomy: keyword, expression, colon, indented block — the same shape as while
  • An elif chain tests top-down and runs only the first true branch — order matters
  • else has no condition — it catches everything that fell through
  • Range checking: because earlier branches already excluded values, elif grade >= 80 means "80–89" without writing the upper bound
  • Indentation decides ownership — a misindented line silently moves in or out of the branch
grades.py
grade = 85
if grade >= 90:
    print("A")
elif grade >= 80:
    print("B")      # first true wins
elif grade >= 70:
    print("C")
else:
    print("Keep working")
# prints: B  (and ONLY B)
Exam trap: reorder those branches (>= 70 first) and an 85 prints C. Always check the order of an elif chain before answering.

Chapter 4⚖️ Comparison & Boolean Operators

Short-circuit: False and … never evaluates the right side; neither does True or ….

booleans.py
age = 25
has_id = False

print(age >= 21 and has_id)   # False
print(age >= 21 or has_id)    # True
print(not has_id)             # True

vowels = "aeiou"
print("e" in vowels)          # True
print("z" not in vowels)      # True
Use parentheses to make mixed and/or expressions unambiguous — and expect the exam to test one without them: and binds tighter than or.

What does this print?

drill_4a.py
temp = 70
if temp > 85:
    print("Hot")
elif temp > 65:
    print("Warm")
elif temp > 45:
    print("Cool")

B — Warm, and only Warm. 70 is not > 85, so the chain falls to temp > 65, which is true — the branch runs and the entire rest of the chain is skipped, even though 70 > 45 is also true. Option C is the classic "every true branch runs" misconception.
Trace Drill · Chapter 4

What does this print?

drill_4b.py
x = 8
y = 3
if x > 5 and y > 5:
    print("both")
elif x > 5 or y > 5:
    print("one")
if not (x == y):
    print("different")
B. 8 > 5 and 3 > 5 → True and False → False; the elif's or needs only one side, so "one" prints. The second if is a separate statement — not part of the chain — and not (8 == 3) is not False → True, so "different" also prints. Watch for a fresh if hiding below an elif chain.

Chapter 5🔁 while Loops & Sentinels

  • Tests at the top, runs the body while true; when false, skips past the whole body
  • The body always finishes its pass — the condition is only checked back at the top
  • Infinite loop = condition can never go false; stop with Ctrl+C
  • Sentinel pattern: input once before the loop, repeat input as the last line of the body (Python has no do-while)
Case matters: a loop testing != "quit" does not stop on Quit.
sentinel.py
user = input("Type quit to exit: ")
while user != "quit":
    print(f"You said {user}")
    user = input("Type quit to exit: ")
print("Goodbye")

You'll use exactly this pattern in tonight's group project.

Chapter 5🔢 Counters — and the Value After the Loop

counter.py
count = 1
while count <= 5:
    print(count)        # 1 2 3 4 5
    count += 1
print(count)            # 6  ← !!

The single most-asked loop question: what is the counter after the loop exits?

The body runs with count at 5, increments it to 6, the test 6 <= 5 fails, and the loop exits — with the counter one step past the last printed value.

Counting down by 5 from 10 while count >= 0 prints 10 5 0 and exits with count == -5. Same principle: one step past.

ShortcutSame as
x += 2x = x + 2
x -= 2x = x - 2
x *= 2x = x * 2
x //= 2x = x // 2

Chapter 5📏 for Loops & range()

  • for item in container: works on lists, strings, tuples, sets (unordered!), and dictionaries (iterates the keys)
  • while vs. for: unknown number of passes (sentinel input) → while; a container or a known count → for
  • range(start, stop, step) — the stop value is never included
  • enumerate(lst) yields (index, value) pairs: for i, v in enumerate(lst):
CallProduces

If a range question is on the exam — and it will be — the wrong answers all hinge on the stop value. Say it with me: stop is not included.

Chapter 5🪆 Nested Loops · break · continue

nested.py
for i in range(1, 4):      # outer
    for j in range(1, 3):  # inner
        print(i, j)
# 1 1 / 1 2 / 2 1 / 2 2 / 3 1 / 3 2
# inner runs COMPLETELY for
# every single outer pass
# total prints: 3 x 2 = 6
  • break exits the loop immediately — but only the innermost loop it sits in
  • continue skips the rest of this pass and jumps to the next iteration
  • Count nested output: outer passes × inner passes — a 9×9 table prints 81 cells
  • Indentation decides which loop a statement belongs to — a print() at the outer level runs once per outer pass
Trace Drill · Chapter 5 — you saw this one in class

What does this print?

drill_5a.py
result = 0
n = 2
while n < 6:
    print(n, end=" ")
    result -= 3
    n += 1
print(result)
B. The loop runs with n = 2, 3, 4, 5 — four passes (when n hits 6, 6 < 6 is false). Four passes × −3 each = −12. Option A includes the 6 (off-by-one on the strict <); option C only counts three passes.
Trace Drill · Chapter 5

What does this print?

drill_5b.py
total = 0
for i in range(2, 10, 3):
    print(i, end=" ")
    total += i
print(total)
C. range(2, 10, 3) produces 2, 5, 8 — then 8 + 3 = 11 is past the stop, so it ends. The stop value (10) is never included, and neither is anything beyond it. Sum: 2 + 5 + 8 = 15.

The user enters 4. What does this print?

drill_5c.py
stop = int(input())    # user enters 4
n = 3
while True:
    print(n, end=" ")
    if n > stop:
        print("Stop")
        break
    n += 1
B. while True looks infinite, but the break is the exit. Trace it: n=3 prints, 3 > 4 false, n→4; prints, 4 > 4 false (strict!), n→5; prints, 5 > 4 true → "Stop", break. The strict > vs >= distinction is exactly the kind of detail the exam hides in plain sight.

Summary⚠️ The Ten Traps That Cost the Most Points

1. input() returns a string — "4" + "3" is "43"
2. = assigns; == compares
3. Indexing starts at 0; last index is len - 1
4. range() never includes the stop value
5. The counter is one step past its last printed value after a loop
6. Only the first true branch of an elif chain runs
7. / always gives a float; // floors; % gives the remainder; int(3.9) is 3
8. Never trust == on floats — 0.1 + 0.2 == 0.3 is False
9. Python is case-sensitive: variables, strings, and "quit" != "Quit"
10. Indentation is meaning — it decides which if/loop owns each line
