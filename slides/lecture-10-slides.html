Section 9.1

Lists: Python's workhorse container

  • A container groups related objects together.
  • A list is a mutable container — it can grow, shrink, and change in place.
  • A list is a sequence — elements keep left-to-right positional order.
  • Elements can be mixed types: strings, ints, floats… even other lists.
list_basics.py
my_list = ["hello", -4.2, 5]     # brackets create a list
also_ok = list("abc")            # list() from any iterable
print(my_list)
print(also_ok)
['hello', -4.2, 5] ['a', 'b', 'c']

Key mental model: the list object holds references to its elements — arrows pointing at objects. This matters enormously in a few slides.

Section 9.1

Accessing elements by index

  • Indices are zero-based: the 1st element is my_list[0].
  • An index can be any expression that evaluates to an integer — my_list[i], my_list[i + 1].
  • A float index is an error, even my_list[0.0].
  • Out-of-range access raises an IndexError — remember Chapter 8? That's an exception you can now try/except.
indexing.py
animals = ["cat", "dog", "bird", "raptor"]
print(animals[0])       # cat
print(animals[2])       # bird
i = 3
print(animals[i])       # raptor
print(animals[4])       # IndexError: list index out of range

Off-by-one alert: the "Nth item" lives at index n - 1. This single fact causes more beginner bugs than anything else in this chapter.

Section 9.1

Aliases vs. copies — the shared reference trap

alias.py — same object!
my_teams = ["Raptors", "Heat", "Nets"]
your_teams = my_teams   # alias

my_teams[1] = "Lakers"
print(your_teams)
['Raptors', 'Lakers', 'Nets']
copy.py — independent
my_teams = ["Raptors", "Heat", "Nets"]
your_teams = my_teams[:]  # COPY

my_teams[1] = "Lakers"
print(your_teams)
['Raptors', 'Heat', 'Nets']

Removing, rearranging, and interrogating

Section 9.3

Iterating: the tracking variable pattern

max_even.py
nums = [3, 5, 23, -1, 456, 1, 6, 83]

max_even = None                  # None = "nothing seen yet"
for num in nums:
    if num % 2 == 0:                          # even?
        if max_even == None or num > max_even:  # new champ?
            max_even = num

print(f"Max even #: {max_even}")
Max even #: 456
  • Each iteration binds the next element to the loop variable num.
  • Why None and not 0? If the list is [-1, 0, -16, -2], starting at 0 gives the wrong answer (-2). None means "no candidate yet" — an honest starting state.
Section 9.3

enumerate() — when you need the index too

by index (works, clunky)
for i in range(len(nums)):
    print(f"{i}: {nums[i]}")
enumerate (Pythonic)
for i, val in enumerate(nums):
    print(f"{i}: {val}")
0: 3 1: 5 2: 23 ...

Rule of thumb: need only values → for x in my_list. Need positions too → enumerate(). Need to modify the list → you'll want the index (Section 9.7 explains why).

Section 9.5

Nested lists: tables in two indexes

tic_tac_toe.py
board = [
    ["X", "O", "X"],   # row 0
    [" ", "X", " "],   # row 1
    ["O", "O", "X"]    # row 2
]
print(board[1][1])   # X
print(board[2][0])   # O
  • A list can hold lists → a 2D table: rows × columns.
  • Access with two index operations: board[row][col].
  • board[1] alone is the entire middle row — a list.
  • len(board) counts rows (3), not all 9 cells.
  • Nesting depth is arbitrary — 3D and beyond work the same way.

Read it inside-out: board[2][0] = "grab row 2, then take its element 0." Same left-to-right evaluation you already know.

Section 9.5

Nested loops walk the whole table

currency.py
currency = [
    [1.00, 5.00, 10.0],   # US dollars
    [0.75, 3.77, 7.53],   # Euros
    [0.65, 3.25, 6.50]    # British pounds
]
for row in currency:          # outer: one row (a list) at a time
    for cell in row:          # inner: one value in that row
        print(cell, end=" ")
    print()                   # newline after each row
1.0 5.0 10.0 0.75 3.77 7.53 0.65 3.25 6.50
  • Need row/column numbers too? Nest enumerate(): for r, row in enumerate(currency): for c, item in enumerate(row): …
  • Common bug: writing the outer list name in the inner loop — you iterate rows twice and never see the cells.
Section 9.6

Slicing: [start : end] — end excluded

slices.py
years = [1992, 1996, 2000, 2004, 2008]
print(years[0:2])    # [1992, 1996]      end index NOT included
print(years[0:-1])   # all but the last: [1992, 1996, 2000, 2004]
print(years[-3:-1])  # [2000, 2004]
  • Slicing always builds a new list — the original is untouched.
  • Negative indices count from the end: -1 is the last element.
  • Same semantics you learned for string slicing in Chapter 7 — one skill, every sequence.
  • Python is forgiving: an end past the last element just means "to the end"; a backwards range gives [] — no crash.
Section 9.6

Stride and the shorthand slices

SliceMeaning[5, 10, 20, 40, 80] →
my_list[a:b]a up to (not incl.) b[0:2] → [5, 10]
my_list[a:b:s]every s-th element[0:5:3] → [5, 40]
my_list[a:]a to the end[2:] → [20, 40, 80]
my_list[:b]start up to b[:4] → [5, 10, 20, 40]
my_list[:]full copy→ new independent list

[:] is the star of this table. Remember the alias trap from 9.1? [:] is your escape hatch — and it's about to save us again in Section 9.7.

Section 9.7

Modifying values: the loop-variable trap

❌ list never changes
for num in nums:
    if num < 0:
        num = 0   # rebinds num,
                  # not the list!
✅ index assignment
for pos in range(len(nums)):
    if nums[pos] < 0:
        nums[pos] = 0  # writes
                       # into list
  • The loop variable is just a name bound to the current element. Reassigning it points the name somewhere new — the list's reference never changes.
  • To change the list, you must assign through an index: nums[pos] = …

Python Tutor moment: this one is invisible in print statements but obvious in the arrow diagram. We'll trace it live.

Section 9.7

Never shrink the list you're walking on

skips_elements.py — subtle logic error
nums1 = [5, 10, 15, 20]
nums2 = [15, 20, 25, 30]

for val in nums1:          # ❌ iterating the list being modified
    if val in nums2:
        nums1.remove(val)  # deletion shifts everything left…

print(nums1)               # 20 survives! Loop skipped over it.
[5, 10, 20] # expected [5, 10] — the 20 escaped
  • Removing an element shifts every later element left one slot — the loop's internal position doesn't know, so the next element gets skipped.
  • The fix costs four characters: iterate over a copy → for val in nums1[:]:

This exact bug appears on real production incident reports. It passes small tests (one deletion) and fails on real data (consecutive deletions). Keep it in mind at 7:50 tonight…

Section 9.8

sort() vs sorted() — in place vs new list

method — modifies original
vals = [150, 47, 500, -37, 0]
vals.sort()
print(vals)
# [-37, 0, 47, 150, 500]
# original is GONE
function — original kept
vals = [150, 47, 500, -37, 0]
in_order = sorted(vals)
print(vals)      # unchanged
print(in_order)  # sorted copy
  • Numbers sort by value; strings sort by Unicode code point — which is why "Z" < "a". Capital letters encode lower than lowercase.
  • Note the return value trap: vals.sort() returns None, so x = vals.sort() gives you None, not a list.
Section 9.8

Steering the sort: key= and reverse=

sort_options.py
names = ["Serena", "venus", "Rafael", "john"]

print(sorted(names))                    # capitals first — surprise!
print(sorted(names, key=str.lower))     # case-insensitive — sane
print(sorted([15, 20, 25], reverse=True))  # highest first
['Rafael', 'Serena', 'john', 'venus'] ['john', 'Rafael', 'Serena', 'venus'] [25, 20, 15]
  • key= takes a function — note str.lower, no parentheses. You're handing over the function itself, not calling it.
  • Any function works: sorted(rows, key=max) sorts 2D rows by their largest value. key=len sorts strings by length.
Section 9.9

Dictionaries: look things up by key, not position

grades.py
my_dict = {
    "Bobby": "A+",
    "Alan": 67,
    10: 5.0
}
print(my_dict["Alan"])  # 67
  • Stores key → value pairs, like a real dictionary maps word → definition.
  • Mutable, like lists — add, change, delete in place.
  • Since Python 3.7, entries keep insertion order.
  • Keys are usually strings or numbers; values can be anything — even lists or other dicts.

When list, when dict? If your data is naturally a numbered sequence → list. If it's naturally a lookup (name → grade, cabin → guest, airport → code) → dictionary.

Dict methods — get() is the one to master

Section 9.11

Iterating: items(), keys(), values()

calories.py
num_calories = {"Coke": 90, "Coke_zero": 0, "Pepsi": 94}

for soda in num_calories:                 # plain loop -> keys
    print(soda)

for soda, cals in num_calories.items():  # the workhorse: pairs
    print(f"{soda}: {cals}")

for cals in num_calories.values():       # just the values
    print(cals)
  • items() yields (key, value) pairs you unpack right in the loop header — same feel as enumerate().
  • These return view objects: memory-light, live-updating, but not indexable. d.keys()[0] is an error — wrap in list() first: sorted(list(d.items())).

Section 9.12 · Optional in zyBooks — required by me

List comprehensions: a loop that returns a list

new_list = [ expression  for  variable  in  iterable ]

the loop way
my_list = [10, 20, 30]
plus_5 = []
for i in my_list:
    plus_5.append(i + 5)
the comprehension way
my_list = [10, 20, 30]
plus_5 = [(i + 5) for i in my_list]

# [15, 25, 35]
  • The brackets are your reminder: a comprehension always builds a new list.
  • Why bother? Less code, fewer append bugs, and the interpreter executes it faster. You'll see these everywhere in professional Python.
Section 9.12

The patterns you'll actually use

greatest_hits.py
# 1. Transform every element
strs = [str(i) for i in [5, 20, 50]]        # ['5', '20', '50']

# 2. Parse user input into ints (you've been writing this as 3 lines)
nums = [int(i) for i in input().split()]

# 3. Per-row work on a 2D list
sums = [sum(row) for row in [[5, 10], [2, 3], [100]]]  # [15, 5, 100]

# 4. Feed one straight into a function
smallest = min([sum(row) for row in table])

Pattern 2 is your new best friend. The read-split-convert loop you've written in every lab since Chapter 5 collapses into one line. Expect it on the final.

Section 9.12

Add an if to filter

[ expression  for  var  in  iterable  if condition ]

filters.py
numbers = [5, 52, 16, 7, 25]

evens     = [i for i in numbers if (i % 2) == 0]   # [52, 16]
negatives = [i for i in numbers if i < 0]          # []
big_x2    = [(i * 2) for i in numbers if i > 20]   # [104, 50]
  • Element only joins the new list when the condition is True.
  • Transform and filter combine freely — expression on the left, filter on the right.
  • Readability rule: if it doesn't fit on one comfortable line, use a regular loop. Comprehensions are a tool, not a religion.

Chapter 9 · Recap

What you can do now

  • Lists — mutable, ordered, zero-indexed containers of references. In-place ops ([i]=, del, methods) vs new-list ops (slices, +).
  • Aliases vs copies — b = a shares one object; b = a[:] makes a real copy.
  • Iteration — for x in list, enumerate() for positions, built-ins (sum, max, any, all) for one-liners.
  • The two loop traps — rebinding the loop variable changes nothing; shrinking a list mid-loop skips elements. Fixes: index assignment; iterate a [:] copy.
  • Sorting — sort() in place vs sorted() new list; steer with key= and reverse=.
  • Dictionaries — key→value lookup, get() for crash-proof reads, items() for iteration.
  • Comprehensions — [expr for x in iterable if cond]: transform + filter in one readable line.
