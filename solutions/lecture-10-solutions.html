B1Guest Manifest Stats (after 9.3)

manifest_stats.py
ages = [34, 8, 71, 45, 12, 67, 29, 88, 15, 52, 41, 6]

print(f"Guests: {len(ages)}  Youngest: {min(ages)}  Oldest: {max(ages)}")
print(f"Average: {sum(ages) / len(ages):.1f}")

minors = 0
for age in ages:
    if age < 18:
        minors += 1
print(f"Minors: {minors}")

for pos, age in enumerate(ages):
    if age >= 65:
        print(f"Position {pos}: age {age}")

print(all(age >= 5 for age in ages))   # stretch — a preview of 9.12!

Debrief talking point: task 1 used built-ins, task 2 used the tracking pattern, task 3 used enumerate() — three iteration tools, one dataset. The stretch answer is secretly a comprehension; call back to it at 7:30 when 9.12 lands.

Expected wrong answers: average printed as an int (missing / len or missing :.1f); minors counted with <= 18 (off-by-one on "under 18"); seniors printed without positions (skipped enumerate).

B2The Standby List Bug (after 9.7)

Buggy program output: Boarding: ['ana', 'cruz', 'dana', 'flo'] — cruz and flo survive because each remove() shifts elements left past the loop's position.

the four-character fix
for guest in standby[:]:       # iterate a COPY, modify the original
    if guest in no_shows:
        standby.remove(guest)   # -> Boarding: ['ana', 'dana']

Stretch: fare = fare * 0.9 rebinds the loop variable — fix is index assignment: for i in range(len(fares)): fares[i] *= 0.9.

B3Deck Directory (after 9.11)

deck_directory.py
directory = {"A101": "Rivera", "A102": "Chen", "B201": "Okafor"}

directory["B202"] = "Dubois"        # same syntax adds…
directory["A102"] = "Vacant"        # …and updates

print(directory.get("A101", "Unassigned"))   # Rivera
print(directory.get("C303", "Unassigned"))   # Unassigned — no crash

for cabin, guest in directory.items():
    print(f"Cabin {cabin} -> {guest}")

for cabin, guest in sorted(directory.items()):   # stretch
    print(f"Cabin {cabin} -> {guest}")

The stretch trick: sorted() accepts the items() view directly and returns a sorted list of tuples, which the for loop unpacks. Watch for teams trying directory.items().sort() — that error is the teachable moment (views aren't lists).

B4Comprehension Conversion Drill (after 9.12 — lightning)

drill answers
doubled    = [n * 2 for n in [3, 7, 11]]
prices     = [float(tok) for tok in input().split()]
comp_fares = [fare * 0.9 for fare in fares if fare != -1]

Judging the race: accept any loop-variable name; #2 must call float() (not int() — they're prices); #3 must have the filter on the right and the transform on the left. Teams that write if fare != -1 for fare in fares have the clauses reversed — most common error.
