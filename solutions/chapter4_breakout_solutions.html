Breakout 1

if / if-else / elif

P1

Trace: What does this code print? Work through it line by line without running it first.

▸ Solution 
num_apples = 5
if num_apples < 7:   # 5 < 7 → True → this branch executes
    print("a")
else:                  # skipped
    print("e")
print("k")             # always runs — outside the if-else
Output a
k
Key concept: The condition 5 < 7 evaluates to True, so the if branch executes and prints "a". The else branch is skipped entirely. Because print("k") is outside and after the if-else block (no indentation), it always runs regardless of which branch fired.
P2

Write: Read an integer into barcode_check_digit. If it equals 7, assign group_id = 1. Otherwise, assign group_id = barcode_check_digit. Print group_id.

▸ Solution 
barcode_check_digit = int(input())

if barcode_check_digit == 7:
    group_id = 1
else:
    group_id = barcode_check_digit

print(group_id)
Test cases Input 7 → 1
Input 3 → 3
Input 0 → 0
Key concept: Use == (equality operator) to compare, never = (assignment). The else branch handles every value that isn't 7 by simply passing the digit through unchanged.
P3

Trace: What is the final value of employee_bonus when num_sales = 7? Which branch executes and why?

▸ Solution
Branch Condition Evaluates to Executes?
if num_sales == 0 7 == 0 False No — skipped
elif num_sales == 1 7 == 1 False No — skipped
elif num_sales == 2 7 == 2 False No — skipped
else (no condition) — ✓ Yes — fires
Result employee_bonus = 10
Key concept: In an if-elif-else chain, Python checks each condition in order and stops at the first one that is True. When num_sales is 7, none of the == checks match, so the else branch is the catch-all that executes. Exactly one branch always fires.
Breakout 2

Ranges & Logical Operators

P1

Fix the bug: This code never prints "Teenager". Find and fix it.

▸ Buggy code 
input_val = int(input())
if (input_val < 13) and (input_val > 19):  # ← BUG: can never be True
    print("Teenager")
else:
    print("Not a teenager")
▸ Fixed code 
input_val = int(input())
if (input_val >= 13) and (input_val <= 19):  # ← FIXED
    print("Teenager")
else:
    print("Not a teenager")
Test cases Input 15 → Teenager
Input 13 → Teenager
Input 19 → Teenager
Input 12 → Not a teenager
Input 25 → Not a teenager
The bug: (input_val < 13) and (input_val > 19) asks for a number that is simultaneously less than 13 and greater than 19 — mathematically impossible. No number satisfies both, so the condition is always False.

The fix: To detect a range, the operators need to close in on the value from both sides: >= 13 (lower bound) and <= 19 (upper bound). Use and to require both conditions to be True at the same time.
P2

Trace: What range of num_passengers values triggers each branch?

▸ Solution 
num_passengers = int(input())
if num_passengers <= 8:       # range: 8 or fewer
    print("Compact van")
elif num_passengers < 15:    # range: 9 – 14  (lower bound implied: > 8)
    print("Medium van")
else:                          # range: 15 or more
    print("Group too large")
Compact van num_passengers <= 8
i.e., 8 or fewer
Medium van 9 – 14
implied lower: > 8; explicit upper: < 15
Group too large 15 or more
catches everything else
Key concept: In a sequential if-elif-else, the lower bound of each branch is implicit — when Python reaches an elif, it already knows all previous conditions were False. So elif num_passengers < 15 doesn't need to also say num_passengers > 8; that's guaranteed by the first condition being False.
P3

Write: Read an integer into lemon_amount. Print "Unrespectable supply" if the amount is fewer than 15 or 25 or more. Otherwise print "Acceptable supply".

▸ Solution 
lemon_amount = int(input())

if lemon_amount < 15 or lemon_amount >= 25:
    print("Unrespectable supply")
else:
    print("Acceptable supply")
Test cases Input 10 → Unrespectable supply (10 < 15 → True)
Input 14 → Unrespectable supply (14 < 15 → True)
Input 15 → Acceptable supply (neither condition True)
Input 20 → Acceptable supply (neither condition True)
Input 24 → Acceptable supply (neither condition True)
Input 25 → Unrespectable supply (25 >= 25 → True)
Input 100 → Unrespectable supply (100 >= 25 → True)
Key concept: This is a range with a gap. The acceptable range (15–24) sits between two unacceptable regions. Use or to trigger when either boundary is violated — if the amount is too low or too high, it's unrespectable. The else catches the acceptable middle.

Common mistake: Students often try and here. lemon_amount < 15 and lemon_amount >= 25 is mathematically impossible (same bug as Problem 1). When detecting two separate regions, always use or.
