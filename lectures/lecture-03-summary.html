🔁 Quick Review

  • Variables are identifiers that point to objects in memory.
  • Identifiers are alphanumeric and case-sensitive; the first character must be a letter or underscore.
  • int objects hold whole numbers; fractional values are truncated.
  • float objects hold decimal / floating-point numbers.
  • Use type() to inspect the type of any variable.
Topic 1

📘 Strings

A string is a sequence of characters enclosed in single or double quotes.

name = "TRISH"

Indexing

Strings use zero-based indexing — the first character is at index 0. Negative indexes count backward from the end.

print(name[0])   # → T  (first character)
print(name[4])   # → H  (fifth character)
print(name[-1])  # → H  (last character, via negative index)
Index out of range: Accessing an index beyond the string length raises an IndexError. Python catches this explicitly — a safety advantage over older languages like C, which would silently read adjacent memory.
Strings are immutable: Once created, a string cannot be modified. To make a change you must create a new string.

Concatenation

The + operator is overloaded for strings — it concatenates rather than adds.

first = "New"
last  = "York"
city  = first + " " + last   # → "New York"

# You cannot concatenate a string with an integer directly — convert first:
room = "Room " + str(5)      # → "Room 5"

F-Strings

Prefix a string with f and place expressions inside {} to embed values directly.

number = 6
amount = 32
print(f"{number} burritos cost ${amount}")  # → 6 burritos cost $32

Topic 2

📋 Lists

A list is a mutable, ordered sequence that can hold items of any type, defined with square brackets.

students = ["Alice", "Bob", "Carlos", "Diana", "Evan"]
print(students[0])   # → Alice
print(students[-1])  # → Evan

Common Operations

students.append("Fatima")      # Add to the end
students.remove("Bob")         # Remove by value
print(len(students))           # Number of items

Lists vs. Strings: Both are indexed sequences, but strings are immutable while lists are mutable — use a list when your collection needs to grow, shrink, or change.

Topic 3

📦 Tuples

A tuple is an immutable, ordered sequence defined with parentheses.

white_house = ("1600 Pennsylvania Ave", "Washington", "DC")
print(white_house[0])   # → 1600 Pennsylvania Ave
  • Tuples cannot be modified after creation — useful for fixed or constant data.
  • Python can optimize tuples internally because it knows they won't change.
  • Tuples print with (); lists print with [].

Named Tuples

Import namedtuple from the collections module to give fields readable names instead of numeric indexes.

from collections import namedtuple

Car = namedtuple("Car", ["make", "model", "price", "seats"])
blazer = Car("Chevrolet", "Blazer", 32000, 8)

print(blazer.make)    # → Chevrolet
print(blazer.price)   # → 32000

List vs. Tuple — When to Use Which

TypeMutable?Use When…
list ✅ Yes Data will grow, shrink, or change
tuple ❌ No Data is fixed (like constants)
Topic 4

🗂️ Dictionaries

A dictionary stores key-value pairs, similar to a real-world lookup table. Defined with curly braces containing key: value pairs.

scores = {}                          # Empty dictionary
scores["Lionel Messi"] = 91
scores["Cristiano Ronaldo"] = 90

print(scores["Lionel Messi"])        # → 91
print(len(scores))                   # → 2
  • Keys are typically strings; values can be any type.
  • Real-world parallel: Redis is a production key-value store used to cache web data rather than repeatedly querying a database.
Topic 5

🔵 Sets

A set is an unordered collection of unique values — duplicates are automatically removed.

# Creating a set from a list (auto-deduplicates)
names = ["Alice", "Bob", "Alice", "Carlos", "Bob"]
unique = set(names)
print(unique)   # → {'Alice', 'Bob', 'Carlos'}
Important gotcha: {} creates an empty dictionary, NOT an empty set. To create an empty set use my_set = set().

Set Operations

A = {1, 2, 3, "gorgon"}
B = {3, "gorgon", 4, 5}

C = A.union(B)           # All unique items from both sets (returns new set)
D = A.intersection(B)    # Only items in both sets (returns new set)
E = A.difference(B)      # Items in A but not in B

Methods: add(), remove(), pop() (removes a random item — order is not guaranteed), clear()

Topic 6

🔄 Type Conversion (Casting)

Python performs implicit conversion in some cases, but often you must convert explicitly.

x = int("42")          # String → Integer
y = float("3.14")      # String → Float
z = str(100)           # Integer → String

Common Gotchas

  • int("3.14") will fail — convert to float first: int(float("3.14"))
  • Strings with commas like "1,000" cannot be converted directly — strip the comma first.
  • input() always returns a string — always cast before doing math.
age = int(input("Enter your age: "))   # Must cast!
Topic 7

💻 Binary & Hexadecimal

Computers store everything in binary (0s and 1s) because circuits have two states: on / off, charged / uncharged.

Hexadecimal (base 16, using 0–9 and A–F) is a compact way to represent binary data — each hex digit represents exactly 4 binary bits.

Binary → Decimal

1101 = 1×8 + 1×4 + 0×2 + 1×1 = 13

Decimal → Binary

Repeatedly divide by 2 and collect remainders (read from bottom to top):
17 ÷ 2 = 8 R1, 8 ÷ 2 = 4 R0, 4 ÷ 2 = 2 R0, 2 ÷ 2 = 1 R0, 1 ÷ 2 = 0 R1 → 10001

Using F-Strings for Instant Conversion

n = 13
print(f"{n:b}")    # → 1101      (binary)
print(f"{n:08b}")  # → 00001101  (8-bit binary with leading zeros)
print(f"{n:x}")    # → d         (hexadecimal lowercase)
print(f"{n:X}")    # → D         (hexadecimal uppercase)
