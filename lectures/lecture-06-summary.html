Exam Technique

✏️ "Playing Computer" — The One Skill the Exam Tests

Almost every question gives you code and asks for the output, the value of a variable, or how many times something runs. The reliable method is the same every time: don't eyeball it — execute it on paper, one line at a time.

  1. Write each variable in a small table on your scratch paper.
  2. Execute one line at a time, updating the table. Never skip ahead or "guess" a loop.
  3. Write down each output the moment a print() runs.
  4. Check the exit condition: what are the variables after the loop ends?
Don't jump to conclusions. Seeing for i in range(1, 5) and instantly saying "runs 4 times" is exactly how the exam catches you. Look at every statement and every condition. Know your operators cold — at least one question uses the modulo operator %.
Chapter 1

🧱 Foundations

  • Every program is input → process → output.
  • Python is interpreted, executing statements one at a time. Languages like C++ are compiled. Expect a definition-style question on this.
  • Python 3 is not backward compatible with Python 2.7. Know the term "backward compatible."
  • input() always returns a string. Wrap it in int() or float() when you need a number.
  • Types: int, float, str — check with type(x).
  • Comments start with #. A multi-line comment uses triple single-quotes '''…''' (this becomes a function's docstring later — not on this exam).
  • Identifiers: letters, digits, underscores; cannot start with a digit; avoid leading double underscores (reserved for Python internals).
  • Two error types: syntax errors (won't parse — e.g., a missing quote) and runtime errors (parse fine, then raise an exception — e.g., ZeroDivisionError, ValueError, NameError). A program that runs but gives a wrong answer is a logic error, not a runtime error.
ch1_print.py
name = input("Name: ")        # always a str
print("Hi", name, sep="--")   # Hi--Ana
print("A", end="")            # no newline
print("B")                    # AB
print() named parameters: sep goes between items (default a single space) and end goes after the line (default "\n", the newline). Setting end="" keeps the next print on the same line; setting sep="" removes the spaces between items.
Chapter 2

🧮 Variables & Expressions

  • = assigns; == compares. A single equals stores a value; a double equals asks a true/false question.
  • Python is case-sensitive. wage, Wage, and WAGE are three different variables — a misspelling silently creates a new one.
  • Compound operators: x += 2 means x = x + 2; likewise -=, *=, //=.
  • Scientific notation: 6.02e23 is 6.02 × 10²³; 1.5e-10 is a very small positive number. The e is the base-10 exponent.
  • Order of precedence follows PEMDAS (parentheses, exponentiation, multiply/divide, add/subtract). It's on the operator cheat sheet — when in doubt, add parentheses.
  • Unicode: ord('A') → 65 (character to number), chr(97) → 'a' (number to character).
  • Float formatting: f"{value:.2f}" displays exactly two decimal places. The stored value keeps full precision; only the display rounds.
Chapter 3

📘 Types: Strings, Containers, and Casting

3a Strings

  • Indexing starts at 0. In "Python", [0] is 'P' and the last valid index is len(s) - 1 (here, 5).
  • Negative indexes count from the end: [-1] is the last character — a quick way to reach the end without computing the length.
  • + concatenates, * repeats: "ab" * 3 → "ababab".
  • Strings are immutable. s[0] = "x" is a runtime error. Lists, dictionaries, and sets can be changed; tuples cannot.
  • An escape sequence is ONE character. len("a\nb") is 3 — internally that's the bytes 65, 10, 66. The \n is for us when we write it; in memory it's a single newline character (decimal 10).

3b The four containers

ContainerSyntaxOrdered?Changeable?Duplicates?
List[1, 2, 3]YesMutableYes
Tuple(1, 2, 3)YesImmutableYes
Dictionary{"a": 1}Keys → valuesMutableKeys unique
Set{1, 2, 3}NoMutableNo

"Changeable/unchangeable" on the cheat sheet are synonyms for "mutable/immutable." Use a tuple for values that should never change after they're set. A set automatically removes duplicates — {1, 2, 2, 3, 3, 3} has length 3. Sets also support union (combine and dedupe) and intersection (keep only what's in both).

3c Type conversion (casting)

ExpressionResult
int("42")42
int("3.5")ValueError — int() can't parse a decimal string
int(3.9)3 — truncates, never rounds
int(float("3.5"))3 — convert to float first, then truncate
str(42)"42" — needed before concatenating a number onto a string
10 / 52.0 — regular division always yields a float
15 // 27 — floor division truncates the remainder
13 % 21 — modulo gives the remainder
Floor division and modulo are partners. // gives the integer (quotient) part; % gives the remainder. Together they let you peel digits off a number: n % 10 reads the last digit, n //= 10 drops it. Also: n % 2 == 1 means n is odd; n % 2 == 0 means even.
Chapter 4

🔀 Branching: if / elif / else

  • An if can stand alone, or have an else, or have one or more elif clauses followed by an optional else. The else has no condition — it's the catch-all.
  • Only the first true branch runs. Once a condition matches, its block executes and the entire rest of the chain is skipped — even if a later condition is also true. This is the most-tested branching idea.
  • Order matters in a range check. When testing "greater than" thresholds, check from highest to lowest; otherwise a low threshold matches first and the logic is wrong.
  • Boolean operators: and needs both sides true; or needs at least one; not flips a value. Python short-circuits — False and … and True or … never evaluate the right side.
  • Membership: x in container / x not in container. For a dictionary, in checks the keys; use .values() to search the values.
temperature.py
temp = 70
if temp > 85:
    print("Hot")
elif temp > 65:
    print("Warm")      # 70 > 65 is true → prints Warm, then STOPS
elif temp > 45:
    print("Cool")      # never reached, even though 70 > 45
else:
    print("Cold")
Strict vs. inclusive matters. In the chain above, the highest value that prints "Cold" is 45 — because 45 > 45 is false, so 45 falls through to the else. If the test were >= 45, the answer would be 44. The exam hides exactly this kind of boundary.
Chapter 5

🔁 Loops

5a while loops & the counter-after-loop trap

A while loop tests its condition at the top, runs the body while the condition is true, and when the condition becomes false it skips the body and continues after the loop. The body always finishes its current pass — the condition is only re-checked back at the top.

counter.py
count = 1
while count <= 5:
    print(count)      # 1 2 3 4 5
    count += 1
print(count)          # 6  — NOT 5
The single most-asked loop question. After the loop, count is 6. The body runs with count at 5, increments to 6, the test 6 <= 5 fails, and the loop exits — but the value 6 is already stored in memory. The expression being false doesn't undo the increment. The counter ends up one step past the last value that passed the test (more if you increment by more than 1).

5b Sentinel pattern

sentinel.py
val = input("Type quit to exit: ")
while val != "quit":
    print(f"You said {val}")
    val = input("Type quit to exit: ")   # repeated at the bottom
print("Goodbye")

Input once before the loop, then repeat the input as the last line of the body (Python has no do-while). It's case-sensitive: Quit won't end a loop testing for quit.

5c for loops, range(), and enumerate()

  • for item in container: walks a list, string, tuple, set (order not guaranteed), or dictionary (you get the keys).
  • while vs. for: unknown number of passes (e.g., reading input until a sentinel) → while; iterating a container or a known count → for.
  • range(start, stop, step) — the stop value is never produced. range(5) gives 0–4; range(2, 10, 3) gives 2, 5, 8 (11 is past the stop).
  • enumerate(lst) yields (index, value) tuples — unpack them with two loop variables: for i, v in enumerate(lst):.

5d Nested loops, break, continue

An inner loop runs completely for every single pass of the outer loop, so the inner body runs outer × inner times. A range(1, 4) outer loop (3 passes) around a range(1, 3) inner loop (2 passes) prints 6 times. "How many times does this run?" is a guaranteed exam question — count each range carefully (remember: stop excluded).

  • break exits the loop immediately — but only the innermost loop it's in.
  • continue jumps back to the top of the loop and re-tests the condition. (A while True loop that always hits continue with no change is an infinite loop.)
On the Exam

❓ The Ternary (Conditional) Expression

This compresses a simple if/else assignment into a single line. There is at least one midterm question on it, so make sure you can read it.

ternary.py
# These four lines...
age = 20
if age >= 18:
    status = "adult"
else:
    status = "minor"

# ...are exactly this one line:
status = "adult" if age >= 18 else "minor"

Read it as: the value before if is used when the condition is true; the value after else is used when it's false. It's called a ternary operator because it has three operands (true-value, condition, false-value) — unlike binary operators such as + (two operands) or the unary not and negation - (one operand). It also works inline, e.g. inside a print: print("pass" if score >= 50 else "fail").

Concept (Not Tested, but Important)

⚠️ Why Floating-Point Numbers Are "Evil"

A computer stores numbers in binary, and many ordinary base-10 decimals have no exact binary representation — just as 1⁄3 = 0.3333… has no exact decimal representation. So floats are stored as very close approximations, not exact values.

float_trap.py
a = 0.1 + 0.2
print(a)              # 0.30000000000000004
print(a == 0.3)       # False  (!)
Never test floats with ==. Because of tiny representation errors, two values that "should" be equal often aren't. Compare with a tolerance (abs(a - b) < 1e-9), or format/round both to a fixed number of decimals first. This won't be tested directly, but it's a real-world rule every Python developer lives by.

A correction from class on rounding. I described Python's display rounding as the "round half up" rule from elementary school (5–9 rounds up). That isn't quite right, and I'll cover this properly in a future lecture. Two things are actually going on: (1) Python's round() and :.2f use banker's rounding (round half to even) — so round(0.5) is 0 and round(2.5) is 2, not 1 and 3. (2) Even that can look "wrong" because the stored value isn't exactly what you typed: 2.675 is really stored as 2.67499999…, so it rounds down to 2.67, not up to 2.68. The takeaway is the same — floats are approximate, so don't rely on exact rounding behavior for anything that has to be exact (like money). None of this is on the midterm.

Drill 7 · digit peeling (modulo + floor division)
drill7.py
number = 12345678
while number > 0:
    print(number % 10, end=" ")   # last digit
    number = number // 10         # drop last digit

Answer: 8 7 6 5 4 3 2 1 — modulo reads each digit, floor division removes it. Forgetting the // 10 makes it an infinite loop.

Summary

🔟 The Traps That Cost the Most Points

  • input() returns a string — "4" + "3" is "43", not 7.
  • = assigns; == compares.
  • Indexing starts at 0; the last index is len - 1 (or -1).
  • range() never includes the stop value — tied for most important rule.
  • The counter ends past the stop value after a loop (usually one more).
  • Only the first true branch of an if/elif chain runs.
  • / always gives a float; // floors; % gives the remainder; int(3.9) is 3.
  • Never trust == on floats.
  • Python is case-sensitive — "quit" != "Quit".
  • Indentation defines blocks — it decides which if or loop owns each line.
Half of the exam's wrong answers are engineered from this list. If you can explain why each one trips people up, you're in good shape.
