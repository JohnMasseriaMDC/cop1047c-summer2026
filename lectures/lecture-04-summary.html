Topic 1

🔀 The if Statement

Until now, every Python program we've written executes line by line from top to bottom. Branching lets the program choose between paths based on a condition — like a railroad switch that routes a train onto one track or another.

1a Simple if

If the condition is true, the indented block executes. If false, it's skipped entirely. The colon at the end of the if line is required — it signals that an indented block follows.

age = 15

if age < 16:
    print("Too young to drive")
    insurance = 0

print("Program continues here regardless")
Indentation is not optional in Python. The indented block under an if must be consistently indented (4 spaces is the standard). A wrong indent is a syntax error.

1b if / else

Add an else clause to handle the case when the condition is false. Exactly one of the two blocks will always execute.

number = int(input("Enter a number: "))

if number % 2 == 0:
    print("Even")
else:
    print("Odd")
Note: The else clause is optional. If you don't need to do anything when the condition is false, you can omit it entirely. A bare else without a body can use pass as a placeholder.

1c if / elif / else

Use elif (short for "else if") to test multiple conditions in sequence. As soon as one branch is true, its block executes and the rest of the elif/else chain is skipped entirely.

years = 25

if years == 1:
    print("Paper anniversary")
elif years == 25:
    print("Silver anniversary")
elif years == 50:
    print("Golden anniversary")
else:
    print("Nothing special this year")
Topic 2

📊 Range Checking with elif Chains

A powerful pattern: by checking ascending threshold values with elif, you can classify a number into a range without using and. Once an earlier condition is false, Python already knows the value is above that threshold.

age = int(input("Enter your age: "))

if age < 16:
    print("Too young")
    insurance = 0
elif age < 25:
    # We know age >= 16 because the first condition failed
    insurance = 4800
elif age < 40:
    # We know age >= 25
    insurance = 2200
else:
    # We know age >= 40
    insurance = 2100

print(f"Annual insurance: ${insurance:,.2f}")
Key rule: For this pattern to work correctly, the threshold values must be in ascending order. If they're not, the ranges will overlap or have gaps and produce wrong results.

Chaining: Python also supports a cleaner way to test if a value falls within a range in a single expression:

age = 16
if 13 <= age <= 19:
    print("Teenager")         # True when 13 <= age <= 19
Topic 3

⚖️ Comparison Operators

All comparison operators return a Boolean value: either True or False. Python evaluates them using a substitution method — it replaces the variable with its value, then evaluates the expression.

OperatorMeaningExample (x = 3)Result
==Equal tox == 3True
!=Not equal tox != 4True
<Less thanx < 4True

<=Less than or equal tox <= 3True
>=Greater than or equal tox >= 3True
Common mistake: Using = (assignment) instead of == (comparison) inside an if condition. These are completely different operations. Python is case-sensitive — variable names must match exactly.
Topic 4

🔗 Boolean Operators

Named after mathematician George Boole, Boolean operators combine two conditions into a single expression. They operate on True/False values — in Python, any zero value is treated as False; any non-zero value is True.

4a and — Both must be true

user_grade = 10

if 9 <= user_grade and user_grade <= 12:
    print("You are in high school")

4b or — At least one must be true

lemon_amount = int(input("How many crates? "))

if lemon_amount < 15 or lemon_amount >= 25:
    print("Unrespectable supply")

4c not — Flips the boolean value

A unary operator (takes only one operand) that inverts True to False and vice versa.

days = 7

if not days > 10:
    print("Less than 11 days")   # days > 10 is False; not False → True
Short-circuit evaluation — Python is smart about Boolean expressions:

AND: If the first operand is False, the result must be False regardless of the second — Python skips evaluating it.
OR: If the first operand is True, the result must be True regardless of the second — Python skips evaluating it.

This matters for compound expressions with many conditions: Python stops as soon as the outcome is determined.

Grouping with Parentheses

Use parentheses to make the grouping of conditions explicit and control operator precedence:

office_num = 1285

# Parentheses group the two AND expressions before the OR
if (office_num >= 100 and office_num <= 500
    or office_num >= 800 and office_num <= 900):
    print("Valid office number")
Topic 5

🔍 Membership Operators: in and not in

Test whether a value belongs to a collection (list, tuple, set, or string). For dictionaries, in tests for a key, not a value.

prices = ["apple", 15, 20]

if 15 in prices:
    print("Found 15")               # True

if 44 not in prices:
    print("44 is not in the list")  # True

# Works on strings too — tests for a substring
if "abc" in "123abcd":
    print("Substring found")        # True (case-sensitive!)

# Inside an if statement
special_list = [0, 7, 42, 99]
special_num = 7
if special_num in special_list:
    print("Special number!")
Strings are case-sensitive. "ABC" in "123abcd" is False because uppercase A, B, C do not match lowercase a, b, c.
Topic 6

🪞 Identity Operators: is and is not

is checks whether two variables point to the exact same object in memory — not just whether their values are equal. Use the id() function to see an object's memory address.

s1 = "hello"
s2 = s1            # s2 points to the SAME object as s1

print(s1 is s2)    # True  — same object in memory
print(id(s1))      # e.g. 140234567890
print(id(s2))      # same number

s2 = "world"       # s2 now points to a DIFFERENT object
print(s1 is s2)    # False
Rule of thumb: Use == to compare values (almost always what you want). Use is only when you need to confirm two variables literally point to the same object — this is rare in beginner programs but important to understand.
Topic 7

↩️ Line Continuation

Long conditions can be split across multiple lines. Python provides two ways to do this:

# Method 1: backslash continuation character
if office_num >= 100 and office_num <= 500 \
   or office_num >= 800 and office_num <= 900:
    print("Valid office")

# Method 2 (preferred): wrap the whole condition in parentheses
# Python knows the statement continues until the closing parenthesis
if (office_num >= 100 and office_num <= 500
    or office_num >= 800 and office_num <= 900):
    print("Valid office")

Method 2 is preferred — parentheses are cleaner, and Python automatically handles the continuation by counting open vs. closed parentheses. The statement continues until the count reaches zero.

  1. Write from scratch: Ask for lemon_amount. Print "Unrespectable supply" if the amount is fewer than 15 or 25 or more. Test with: 10, 15, 20, 25.
