Topic 1

🔁 The while Loop

A loop executes a block of code repeatedly. Think of the textbook's example: driving a child around the block until they fall asleep — while the child is awake, keep driving; once asleep, continue on home. That test-then-repeat structure is exactly what a while statement does.

1a Syntax: just like if, but it repeats

A while statement is the reserved word while, an expression, a colon, and an indented block — the same anatomy as last week's if statement. The indented block is called the body of the loop.

while_basics.py
# Doubles value each pass: prints 1, 2, 4 (one per line), then Done
value = 1
while value < 6:
    print(value)
    value = value * 2
print("Done")
Key idea: An if block runs at most once. A while body runs as long as the expression is true — the expression is re-tested at the top of every pass. When it's false, Python skips the whole body and continues with the first statement after the loop.

Anything that works as an if expression works here too: comparisons, compound conditions with and / or / not, and so on. Python also short-circuits: in a and b, if a is false it never evaluates b (false and anything is false); in a or b, if a is true it never evaluates b (true or anything is true).

Common mistake: The body does not stop mid-pass the instant the condition becomes false. The whole body finishes, control loops back to the top, and then the expression is re-tested.

1b Side trip: print() and its end= parameter

By default, print() ends every line with a newline (\n) because the named parameter end defaults to "\n". Override it to keep output on one line:

print_end.py
print("hello")            # hello, then a newline
print("hello", end="")    # hello, no newline
print(value, end=" ")     # value followed by a space, same line

We'll learn more about named vs. positional parameters when we cover functions, but end= is worth knowing now — it shows up constantly when printing inside loops.

Topic 2

♾️ Infinite Loops

If the loop's condition can never become false, the loop runs forever — an infinite loop. (Apple's old headquarters address was literally One Infinite Loop — programmer humor.) The symptom is a program that just keeps running.

  • To stop one: you need something external to the program — usually Ctrl+C at the keyboard, which sends an interrupt.
  • To debug one: add temporary print() statements inside the loop to see what the variables are actually doing. You can't fix what you can't see.
  • They're not always bugs: programs reading from sensors or communication links may intentionally loop forever, waiting for data. But in the programs we write for this class, an endless run almost always means your condition never goes false.
Topic 3

🚦 The Sentinel Pattern: Loop Until the User Says Stop

A sentinel value is a special input that signals "stop looping." This pattern — input once before the loop, then repeat the same input as the last statement in the body — is one you'll use over and over, including in next week's temperature-conversion group project:

sentinel_quit.py
# Echo what the user types until they type quit
user = input("Type quit to exit: ")
while user != "quit":
    print(f"You said {user}")
    user = input("Type quit to exit: ")   # repeated at the bottom
print("Goodbye")
Note: Python is case-sensitive — typing Quit, QUIT, or q will not end this loop. Only an exact quit matches.
Why the duplicated input line? Some languages have a do-while loop that always runs the body at least once. Python doesn't — so the standard pattern is to repeat the input statement: once before the loop to get the first value, and again at the bottom of the body to get the next one.

This little program is essentially the front end of a chatbot: replace the print with a call to an AI API and you'd have a working chat loop. Also note the f-string — f"You said {user}" evaluates the variable (or any expression!) inside the curly braces and inserts its value.

Topic 4

🔢 Counting with Loops

4a The counter pattern

counter.py
count = 1
while count <= 5:
    print(f"count is {count}")
    count += 1          # same as: count = count + 1
print(f"after the loop, count is {count}")
Midterm alert: What's the value of count after this loop? It's 6, not 5. The body runs with count at 5, increments it to 6, loops back, the test 6 <= 5 fails, and the loop exits with count == 6. "What's the variable's value when the loop completes?" is a classic exam question — trace it on paper.

Counting down works the same way. Starting at 10, testing count >= 0, and doing count -= 5 prints 10 5 0 — and leaves count at -5 after the loop.

4b Compound operators (review)

ShortcutEquivalent to
x += 5x = x + 5
x -= 3x = x - 3
x *= 2x = x * 2
x /= 4x = x / 4 (regular division)
x //= 4x = x // 4 (floor division)
Topic 5

📦 The for Loop: Iterating Over Containers

5a for ... in with a list

for_basics.py
names = ["Kai", "Sam", "Alex"]
for name in names:          # name takes each item, in order
    print(f"Hi {name}")
print("Done")

The for ... in statement pulls each item out of the container, one per pass, and assigns it to the loop variable. Naming convention: the container is plural (names), the loop variable is singular (name). The language doesn't care — but humans reading your code do.

This works with any container:

  • Lists — items in order
  • Strings — character by character (for ch in "abcdefg":) — remember, a string is essentially a list of characters
  • Dictionaries — you iterate over the keys, then use each key to look up its value (this lookup is guaranteed safe, since the keys came from the dictionary itself)
  • Tuples — same as lists
  • Sets — works, but order is not guaranteed; sets are unordered (their advantage is automatic de-duplication)
for_dict.py
channels = {"MTV": 35, "CNN": 28}
for name in channels:                 # iterates over the KEYS
    print(f"{name}: {channels[name]}")
# MTV: 35
# CNN: 28

The reversed() function gives you the items of a list in reverse order — for name in reversed(names): — far cleaner than managing a count-down index by hand.

5b Why for beats a manual index

You can walk a list with a while loop and an index variable — but it takes more lines and gives you more ways to make a mistake (initialize wrong, test wrong, increment wrong, index out of range):

manual_index.py
# The hard way — 3 extra things to get right
index = 0
while index < len(names):
    print(f"Hi {names[index]}")
    index += 1

# The Python way — nothing to get wrong
for name in names:
    print(f"Hi {name}")
Key idea: Take advantage of the features of the language so you don't accidentally introduce errors doing things the manual way. A lot of Python programming is just: put your items in a list, then for ... in over it.

5c while vs. for — which one when?

Use a while loop when…Use a for loop when…
You don't know how many times you'll iterate — e.g., reading user input until a sentinel like quit or -1 You're iterating over a container (list, string, dictionary, tuple, set)
The loop runs "until something happens" You want to run something an exact number of times — for i in range(100):
Topic 6

📏 The range() Function

range() returns an iterator that produces a sequence of integers — perfect fuel for a for loop. It takes up to three values: range(start, stop, step).

CallProducesWhy
range(5)0, 1, 2, 3, 4start defaults to 0, step defaults to 1
range(5, 10)5, 6, 7, 8, 9start at 5; stop value not included
range(0, 10, 2)0, 2, 4, 6, 8all the even numbers below 10
range(5, 10, 3)5, 88 + 3 = 11 is past the stop, so it ends
range(10, 0, -1)10, 9, … , 1negative step counts down; stops before 0
The #1 thing to remember: the stop value is never included. range(5) runs a loop 5 times with values 0–4. To print multiples of 3 from 3 through 30, you need range(3, 31, 3) — the 31 is what lets the 30 through.

6a range(len(...)) — when you need the index

range_len.py
foods = ["pizza", "sushi", "tacos", "pasta", "ice cream"]
for i in range(len(foods)):
    print(f"{i + 1}: {foods[i]}")
# 1: pizza
# 2: sushi  ... and so on

Why i + 1? List indexes start at 0, but humans count from 1. We use i to index into the list and i + 1 only for display. And because we used len(foods) instead of hard-coding a number, this code works for a list of any size — including an empty one. Never hard-code what the language can figure out for you.

Topic 7

🪆 Nested Loops

Just like nested if statements, loops can contain loops (and ifs can be inside loops, and vice versa). Terminology: the enclosing loop is the outer loop, the enclosed one is the inner loop. The inner loop runs completely for every single pass of the outer loop.

times_table.py
# A multiplication table, using \t (tab) to line up the columns
for i in range(1, 10):
    for j in range(1, 10):
        print(f"{i} x {j} = {i * j}", end="\t")
    print()        # end the row

The \t tab character advances to a fixed tab stop, so columns line up even when products grow from one digit to two. The inner print() with no arguments just emits the newline that ends each row — notice its indentation puts it in the outer loop's body, so it runs 9 times, while the f-string print runs 81 times. Watch your indentation levels — they decide which loop a statement belongs to.

The textbook's two-letter .com domain generator nests two while loops over the alphabet using ord() (letter → number, ord("a") is 97) and chr() (number → letter) to step from a to z — producing all 26 × 26 = 676 combinations.

Topic 8

⏭️ break and continue

StatementWhat it does
breakExits the loop immediately and completely. In nested loops, it only exits the innermost loop it's in — to leave the outer loop too, set a flag (sentinel) variable, break, then test the flag in the outer loop.
continueSkips the rest of the current pass and jumps to the loop's next iteration (the condition is re-tested as usual).
continue_odds.py
# Prints only the odd numbers 1 and 3
for i in range(5):
    if i % 2 == 0:
        continue        # skip the evens
    print(i)
Use them sparingly. Someone reading your code may not notice a buried break or continue. Often you can restructure the condition instead (e.g., test for odd and print, rather than test for even and skip). There are valid use cases — just make them obvious.
Topic 9

🔖 enumerate(): Index and Value Together

Like range(), enumerate() returns an iterator — but it yields a tuple of (index, value) for each item. The two loop variables are filled in by unpacking that tuple:

enumerate_demo.py
scores = [4, 8, 10]
for index, value in enumerate(scores):
    print(f"Index {index} holds {value}")
# Index 0 holds 4
# Index 1 holds 8
# Index 2 holds 10

This is the Pythonic way to get an item's position while iterating. It beats calling list.index(value), which returns only the first match and silently fails you when the list has duplicates. Knowing the index matters because lists are mutable — you can use the index to change an item in place. (You can't do that with strings; strings are immutable.)

Optional (not on the midterm): loops can have an else clause that runs only if the loop finished without hitting a break. It's from Chapter 12 and included tonight just for completeness.

🧪 Breakout Exercises

Breakout 1 · While & For Fundamentals

Three challenges

  1. Multiples of 3: Print all multiples of 3 from 3 to 30 — first with a while loop (no range()), then again with for ... in range(). Hint: the stop value isn't included, so you need 31.
  2. Favorite foods: Create a list of your 5 favorite foods. Print each item; then print each as a numbered line (1: pizza, 2: hot dogs, …) using range(len(...)) and i + 1 for display.
  3. Sum and average: Loop exactly 5 times asking the user for a number; print the total and average. Then write the variation: a while loop that keeps reading numbers until the user enters -1, then prints how many numbers were entered and their average. Defensive bonus: guard against dividing by zero if the user enters -1 immediately.
Breakout 2 · Nested Loops & Incremental Development

Three challenges

  1. Seating chart: Ask the user for a number of rows and columns; print a chart with numbered rows and lettered columns using a nested loop.
  2. Incremental sum: Build it in stages — (1) loop 5 times printing i; (2) replace the print with an accumulator that sums the numbers; (3) instead of hard-coding 5, ask the user for the upper limit. This is incremental development: get a small piece working, then extend it.
  3. Challenge: Extend the seating chart so only the even-numbered rows print. Two ways: test row % 2 == 0 inside the loop, or adjust the range to start at 2 and step by 2.
