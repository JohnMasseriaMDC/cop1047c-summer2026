Topic 1

🧩 Why Write Functions?

We've used functions all semester — print() and input() are functions. Tonight is about writing our own. The core reasons:

  • Reuse. A loop repeats code in one spot; a function lets you run the same code from anywhere, as many times as you like, with different inputs each time.
  • Single source of truth. If you copy-paste a temperature formula five times, you've created five chances to introduce a typo (one wrong operator and that result is silently wrong). Put the formula in one function and every caller gets the correct, identical logic.
  • Testing. Once a function is thoroughly tested, you can trust it — call it with any valid input and rely on the result.
  • Decomposition (divide and conquer). Break a big problem into small, independently testable pieces, then assemble them. Think of functions as Lego blocks you snap together.
Topic 2

🛠️ Defining a Function: def and return

f_to_c.py
def f_to_c(f):
    """Convert a Fahrenheit temperature to Celsius."""
    c = (f - 32) * (5 / 9)
    return c

c1 = f_to_c(32)     # 0.0
c2 = f_to_c(212)    # 100.0

The anatomy: the def keyword (short for define), the function name (an identifier — same rules as any variable: letters, digits, underscores; can't start with a digit; case-sensitive), parentheses holding the parameter list, a colon, and an indented body. The body is a block, just like the body of a loop or an if.

  • return hands a value back to the caller and exits the function. The whole function call becomes that returned value: after c1 = f_to_c(32), the call is replaced by 0.0.
  • Define before you call. Python must have read the def before it reaches a call to it. Standard program layout: imports at top, then all function definitions, then the main program that uses them.
  • Parentheses matter: (f - 32) is parenthesized so the subtraction happens before the multiplication — without it, order of operations would compute the wrong result.
Docstring: a triple-quoted string as the first line of the body becomes the function's documentation. It's what help(f_to_c) displays. Best practice is to write one: say what the function does and what its parameters mean.
You can return an expression directly. There's no need for a throwaway variable — return (f - 32) * (5 / 9) works just as well as computing c first and returning it. def compute_square(n): return n * n is a complete function.
Topic 3

📥 Parameters vs. Arguments

The vocabulary distinction matters and may be tested:

TermWhereExample
ParameterIn the definition — a placeholder namedef calc(pizza_diameter):
ArgumentIn the call — the actual value passedcalc(12) — 12 is the argument

When the function is called, Python creates a temporary local variable for each parameter and fills it with the matching argument: calc(12) sets pizza_diameter to 12 for the duration of that call. A function can take several parameters, matched left-to-right:

two_params.py
import math

def calc_pizza_volume(diameter, height):
    radius = diameter / 2
    area = math.pi * radius * radius
    return area * height          # cylinder: area x height

v = calc_pizza_volume(12, 0.3)    # diameter=12, height=0.3
Naming convention: name functions with a verb and underscores — calc_pizza_volume, convert_to_celsius. The camelCase style (capital-letter "humps") works syntactically but isn't the recommended Python style for function names.
Don't put an expression in the parameter list of a definition. def my_func(x + 5): is invalid — the definition expects plain parameter names. To add 5 to the input, do it in the body: x = x + 5.
Topic 4

🔭 Variable Scope: Local vs. Global

  • Local variables are created inside a function (including its parameters). They exist only while the function runs and vanish when it returns — their memory is reused. You can't reference them from outside.
  • Global variables are created outside any function. They're visible everywhere, including inside functions.
  • Two variables with the same name — one inside a function, one outside — are genuinely different variables (you can prove it with id()). The parameter name doesn't have to match the argument's variable name.
scope.py
employee_name = "N/A"        # global

def get_name():
    employee_name = input("Name: ")   # creates a NEW local!

get_name()
print(employee_name)         # still "N/A" — global unchanged
The assignment trap: assigning to a name inside a function creates a local variable, even if a global of the same name exists — so the global is left untouched. To actually modify the global, declare global employee_name as the first line of the function. (Constants are the usual reason to read globals; use the global keyword sparingly.)
Constants convention: Python has no true constants, so by convention we name them in ALL_UPPERCASE (e.g. CENTIMETERS_PER_INCH = 2.54) as a signal "don't change this." Nothing stops a program from changing it — the uppercase is a human warning, not an enforced rule.
Topic 5

🕳️ Void Functions and None

A function with no return (or a bare return with no value) is a void function. It still returns something — the special value None. Python adds an implicit return None at the end of every function that doesn't return otherwise.

none_trap.py
def miles_to_km(miles):
    km = miles * 1.60934        # forgot to return!

result = miles_to_km(5)
print(type(result))             # <class 'NoneType'>
print(f"{result:.2f}")          # TypeError — can't format None
Forgetting return is a top mistake. The function runs, computes the answer, and throws it away — returning None. Using that None in arithmetic or an f-string raises a TypeError. A void function (like one that only prints a menu) is fine; the bug is forgetting return when you needed the value.

Return a status signal. Many functions return 0 for success and a non-zero or -1 for an error (a sentinel) so the caller can check whether things worked. Error codes are often used as an index into a list of error messages — remember to subtract 1 if the codes start at 1 but list indexes start at 0.

📦 Using Modules

Two ways to pull in library code, shown with math:

Import styleHow you use itTrade-off
import mathmath.pi, math.pow(2, 3)Brings in the whole module — larger, but everything's available
from math import pipi (no prefix needed)Brings in just that one name — smaller, and you skip the math. prefix

Imports go at the top of your program. Many hardware devices ship with a module too — the classroom display's library is imported the same way, which is what makes driving the hardware easy.

Topic 7

🎁 Returning Multiple Values

List the values after return, separated by commas, and Python bundles them into a tuple. The caller unpacks the tuple into the matching number of variables.

multi_return.py
def cm_to_us_height(cm):
    inches = cm / 2.54
    feet = inches // 12          # whole feet
    inches = inches % 12         # leftover inches
    return feet, inches          # a tuple of two values

feet, inches = cm_to_us_height(193.04)   # unpack: 6, 4
one_var = cm_to_us_height(193.04)        # a tuple (6.0, 4.0)
The variable count must match. If a function returns 3 values, you must unpack into exactly 3 variables — asking for 2 (or 4) raises a ValueError about unpacking. Assign to a single variable and you simply get the whole tuple. This is the same floor-division/modulo digit-splitting idea: // 12 gives whole feet, % 12 gives leftover inches.
Topic 8

🦆 Dynamic Typing, Duck Typing & Operator Overloading

  • Dynamic typing: a variable's type can change as you reassign it — km = 10 (int), then km = "ten" (str). You never declare types; Python infers them. (Statically-typed languages lock a variable to one type.)
  • Duck typing: "if it walks like a duck and quacks like a duck, it's a duck." Python cares what a value can do, not what it's formally declared as.
  • Operator overloading (polymorphism): the same operator behaves differently by type. + adds numbers, concatenates strings, and joins lists; * multiplies numbers but repeats strings ("ha" * 3 → "hahaha").
overload.py
def add(x, y):
    return x + y

print(add(5, 7))                # 12   (integer add)
print(add("Tora", "Bora"))      # ToraBora  (concatenate, no space)
print(add([1, 2], [3, 4]))      # [1, 2, 3, 4]  (list join)
print(add(5, "x"))              # TypeError — int + str not allowed
Mixing an int and a float works — the int is coerced to float. But mixing a number and a string is a TypeError; you must convert explicitly with str() or int() first.
A peek under the hood (optional): when an f-string formats a value, it calls that value's built-in __format__ method. Floats, ints, and strings all have one, which is why f"{x:.2f}" works on them — and why it fails on None, which has no such method. This is object-oriented machinery we'll explore later; not on any exam.
Topic 9

🚧 Incremental Development: Stubbing Out Functions

While building a program, you often want to call a function before you've written it. Three ways to "stub" it:

StubBehavior
passDoes nothing; the function returns None. Lets the program run, but using the result errors.
print("FIX ME"); return -1Prints a reminder and returns a sentinel error value the caller can detect.
raise NotImplementedErrorStops the program with a clear error if the unfinished function is ever called — the safest stub.

The third is best when you want to guarantee the function isn't used before it's finished, because it can't silently return a wrong answer.

Topic 10

🔁 Recursion (Light Touch)

A function can call itself — that's recursion. Every recursive function needs two parts:

  • Base case: an if at the top that stops the recursion (you've reached the simplest input).
  • Recursive case: a call to itself with a "smaller" input that moves toward the base case.
recursion.py
def countdown(count):
    if count == 0:           # base case
        print("GO")
        return
    print(count)
    countdown(count - 1)     # recursive case — moves toward 0

def factorial(n):
    if n == 1:
        return 1
    return n * factorial(n - 1)   # 6! = 720
If the recursive call doesn't approach the base case, you get infinite recursion. Python caps recursion depth at about 1000 calls and then raises a RecursionError. countdown only works for a positive start; a negative input never reaches 0.
What's testable: just the concept — that a recursive function needs a base case to stop, and that the recursive call must lead toward it. Factorial and the Fibonacci sequence are the classic examples (Fibonacci is recursively defined: each number is the sum of the two before it, and it even shows up in nature, like spiral shells).
